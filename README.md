# IoT Based Smart Reminder System

Have you ever set an important reminder on your phone only to miss it because the device was in silent mode, left in another room, or buried under tons of notifications? While smartphones have made reminders convenient, they are not always used in every situation, especially for elderly people, students, or professionals with busy schedules.

To solve this problem, we have designed an **IoT-based Smart Reminder System** that acts as a dedicated personal reminder assistant. Instead of relying solely on a mobile phone, the system provides both visual and voice notifications through dedicated hardware. Users can easily create reminders from a web browser over Wi-Fi, and at the scheduled time, the ESP32 automatically displays the message on an e-paper display and announces it through a speaker. Also, the system will retain the last displayed content even when power is removed. This means that if there is a power outage after a reminder has been announced, the most recently displayed reminder will continue to remain visible on the screen without consuming any power.

## Features
- **Web-Based Interface**: Create, view, and test reminders easily via a local web server hosted on the ESP32.
- **E-Paper Display**: Retains the last displayed reminder even when the power is off, saving energy.
- **Voice Announcements**: Generates text-to-speech (TTS) audio files and plays them through a speaker.
- **Standalone System**: Eliminates the need to constantly check a smartphone.
- **Up to 10 Reminders**: Capable of storing and managing up to 10 active reminders.

## Workflow

1. **System Initialization**: Connects to Wi-Fi, mounts LittleFS, initializes the e-paper display and audio modules, and synchronizes time via NTP.
2. **Web Interface Access**: Starts an embedded web server accessible via a local IP address in any web browser.
3. **Creating a Reminder**: User inputs the text and time. The ESP32 saves the reminder and generates a TTS MP3 file stored in LittleFS.
4. **Time Monitoring**: Continuously compares the current NTP time with scheduled reminders in the background.
5. **Reminder Triggering**: Updates the e-paper display with the text and plays the voice announcement via the ADA4841-2 and PAM8403 amplifier.
6. **System Reset**: Marks the reminder as played for the day to avoid repeating playback.

## Schematic & Hardware
The core of the system is the ESP32, communicating with the e-paper display via SPI interface using DIN, CLK, CS, DC, RST, and BUSY pins. The DAC output from the ESP32 goes through a low-pass filter and the ADA4841-2 op-amp for signal conditioning, before being amplified by the PAM8403 to drive the speaker.

## Troubleshooting

| Problem | Possible Cause | Solution |
|---------|---------------|----------|
| **E-paper display blank** | Incorrect SPI wiring or power connection | Check DIN, CLK, CS, DC, RST, BUSY connections; ensure 3.3V power. |
| **No audio output** | Wrong wiring or amplifier issue | Check PAM8403 and speaker wiring; ensure 5V supply to amplifier. |
| **Distorted/noisy audio** | Incorrect filter/op-amp connection | Verify C1-C5, R1-R4, and ADA4841-2 circuit connections. |
| **Speaker output very low** | Low input signal or wiring issue | Check the audio path and verify PAM8403 input and speaker connections. |
| **ESP32 keeps restarting**| Insufficient power supply | Use a stable power source capable of supplying adequate current. |
| **Compilation/upload errors**| Incorrect board or port selection | Select the correct ESP32 board and COM port in Arduino IDE. |

## FAQ

- **How are reminders added?** Via the web interface hosted on the ESP32's IP address.
- **How many reminders can be stored?** Up to 10 at a time. Delete old ones to add new ones if the limit is reached.
- **Does the system require an internet connection?** Yes, for Wi-Fi, NTP synchronization, and generating TTS audio files.
- **Why use an e-paper display?** It consumes power only during screen updates and retains content even without power, saving energy.
- **What is the purpose of the ADA4841-2 op-amp?** It conditions and filters the audio signal from the ESP32, reducing switching noise and improving quality.
- **Why is the PAM8403 amplifier required?** The ESP32 cannot drive a speaker directly; it amplifies the audio for clear output.
- **Can reminders be edited?** Currently, you need to delete and recreate the reminder to edit it.

## Conclusion
This dedicated hardware device provides clear voice announcements and visual reminders through an e-paper display. It is perfect for elderly individuals, those with memory difficulties, study environments, or situations where phone usage is restricted.
